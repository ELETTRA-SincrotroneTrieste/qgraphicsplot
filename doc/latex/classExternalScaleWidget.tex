\doxysubsection{Detailed Description}
\begin{DoxyParagraph}{Structure}
The \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} is a Q\+Widget that draws a scale with ticks and numeric values. It can be horizontal or vertical. Implements \doxyref{Axis\+Change\+Listener}{p.}{classAxisChangeListener} in order to receive notifications when the maximum and minimum values of the scale change and Plot\+Event\+Listener in order to be notified when the \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} area changes, for instance when the user scales the view to magnify a portion of the scene. Nonetheless, it is not essential to install an \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} if you intend to use this class without a \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. Actually, there are slots equivalent to the methods implemented from the \doxyref{Axis\+Change\+Listener}{p.}{classAxisChangeListener} and Plot\+Event\+Listener interfaces. These slots are area\+Changed and scroll\+Changed. The \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} needs to know when the area of the Q\+Scroll\+Area contents changes and when the value of the scrollbars change, so that it can recalculate the values to display in the correct points.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Note}
You can use the \doxyref{Horizontal\+Scale\+Widget}{p.}{classHorizontalScaleWidget} and \doxyref{Vertical\+Scale\+Widget}{p.}{classVerticalScaleWidget} convenience classes to directly instantiate horizontal and vertical scales respectively. The Qt Desiger plugin uses those classes.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Align scales with the Plot\+Scene\+Widget or other scrolling areas}
It is very important that the height of a vertical scale is equal to the height of the widget it is associated to, and that the width of a horizontal scale is equal to the widget\textquotesingle{}s width. Using the Qt4 designer, just place and align the widgets, and then use a grid layout. The Q\+Grid\+Layout will ensure that the widgets are all scaled together.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Example}
Suppose you have an image from a camera and you want to use an external scale to identify pixel positions inside the image. In this case the \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} is not used in association with a \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} and thus we use signal/slot connections to tell the scale that the image area or the values of the scroll bars have changed.
\end{DoxyParagraph}

\begin{DoxyCode}{0}

\begin{DoxyParagraph}{Inverted scales}
The maximum must always be greater than the minimum, but if you want the scale to draw the values starting with the minimum on the rigth (for a horizontal scale) or the top (for a vertical scale), just call set\+Inverted on the \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget}.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Tick drawing inverted}
If a vertical scale is placed on the right of the display area, then you may want to call set\+Tick\+Drawing\+Inverted in order to place ticks before numeric values. The same goes for {\itshape top} placed horizontal scales.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Labels format}
The format of the labels is not automatically calculated as in the \doxyref{Scale\+Item}{p.}{classScaleItem}. It is {\itshape \%.1f} by default, and you must call the set\+Format or use the format option in order to change it.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Origin position percentage}
A property called origin\+Percentage is introduced in order to tune the scale positioning. A value of 0.\+5 for instance tells the scale that its {\itshape center of gravity} should be in the middle of its length. A tick is guaranteed to be positioned at the half of the length of the scale and, according to the number of ticks, some extra empty space at the beginning or end of the scales might appear, without ticks. Actually, if used in conjunction with a \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}, the origin of its axes is normally placed at the center of the plot. To align the \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} with the grid of the \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}, you should tune the origin\+Percentage property so that its value is 0.\+5. The \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} itself has a property to change the axis origin position percentage\+: on the \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} an equal value should be set with set\+Origin\+Percentage.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Orientation}
The horizontal property is a shortcut for set\+Orientation. It switches a scale from the horizontal to the vertical layout, being the parameter true or false respectively.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Tick length}
The tick\+Len property assigns a length (in pixel coordinates) to the scale ticks.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Tick step length}
The tick step length is used to calculate the number of ticks on the scale and where the ticks have to be placed. The value is expressed in {\itshape scale coordinates}. For instance, if your scale spans from 0 to 100 and you specify a tick\+Step\+Len of 10, you will have ten ticks with ten values drawn on the widget.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Label Rotation}
The label\+Rotation property is able to rotate the labels on the scale. Up to now, it is only possible to rotate {\itshape x axis} labels.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Notes on widget geometry.}
Whenever one of the minimum, maximum, label\+Rotation, format or tick\+Len properties changes, the minimum\+Size\+Hint for the widget changes, and the update\+Geometry() is internally called by the setter methods. In this way you can safely insert the widgets into a Q\+Layout without worrying about geometry changes (resizes, changes in tick len or format...). The minimum\+Size\+Hint takes into account the font in use, naturally.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Example}
This example shows how to use two \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} scales with a \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. As said before, \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} implements two interfaces aimed at receive \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} notifications in matter of geometry change and \doxyref{Scale\+Item}{p.}{classScaleItem} notifications in matter of minimum and maximum value changes. All you need to to is to install each \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} on the plot and on the scale items of the plots as in the example below.
\end{DoxyParagraph}

\begin{DoxyParagraph}{Personalized labels}
You can implement a \doxyref{Scale\+Label\+Interface}{p.}{classScaleLabelInterface} and install it on an \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget}. The \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} will call the \doxyref{Scale\+Label\+Interface\+::label}{p.}{classScaleLabelInterface_a1c68e07c44a7aed5c90b72ad46c80a4c} implementation with the value corresponding to the tick being drawn. Your \doxyref{Scale\+Label\+Interface\+::label}{p.}{classScaleLabelInterface_a1c68e07c44a7aed5c90b72ad46c80a4c} implementation has to convert the raw value passed by the \doxyref{External\+Scale\+Widget}{p.}{classExternalScaleWidget} into a convenient label according to your needs. For instance, a \doxyref{Time\+Scale\+Label}{p.}{classTimeScaleLabel} can be used to convert unix timestamps into a date/time string. 
\end{DoxyParagraph}
