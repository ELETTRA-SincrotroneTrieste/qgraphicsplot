\begin{DoxyParagraph}{Plot\+Scene\+Widget}
\doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} is a Q\+Graphics\+View with a Q\+Graphics\+Scene where Q\+Graphics\+Item items can be placed and displayed. In particular, \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} is a {\itshape plot} where {\itshape curves} can be drawn.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Axes}
A \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} has by default a couple of axes, the x and y axes, and their origin is by default in the center of the scene. If you want to change the axes position, see \begin{DoxyItemize}
\item \doxyref{Plot\+Scene\+Widget\+::set\+Origin\+Pos\+Percentage}{p.}{classPlotSceneWidget_aea844334c4b2e71ff6eba2f57cd562f9} \item \doxyref{Plot\+Scene\+Widget\+::set\+Default\+X\+Axis\+Origin\+Pos\+Percentage}{p.}{classPlotSceneWidget_a0c9b92b6a5aca6da0fa85b9f9b7aca32} and \item \doxyref{Plot\+Scene\+Widget\+::set\+Default\+Y\+Axis\+Origin\+Pos\+Percentage}{p.}{classPlotSceneWidget_a759d14c7e662ce3afabebd7dcf167d9c}.\end{DoxyItemize}

\end{DoxyParagraph}
\begin{DoxyParagraph}{Reducing the plot area inside the scene.}
It is possible to reduce the area occupied by the plot by means of the plot\+Area\+Percentage\+Width, plot\+Area\+Percentage\+Height, plot\+Area\+Top\+Left\+X\+Percentage, plot\+Area\+Top\+Left\+Y\+Percentage properties.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Curves}
Each curve is represented by a \doxyref{Scene\+Curve}{p.}{classSceneCurve}. A \doxyref{Scene\+Curve}{p.}{classSceneCurve} contains the data of the curve. The \doxyref{Scene\+Curve}{p.}{classSceneCurve} is not responsible for drawing itself. Each \doxyref{Scene\+Curve}{p.}{classSceneCurve} must be given a name and added to the plot by means of \doxyref{Plot\+Scene\+Widget\+::add\+Curve}{p.}{classPlotSceneWidget_ad84fa604445007630e530ad65b24b828} ~\newline
 The painting takes place by a \doxyref{Curve\+Item}{p.}{classCurveItem} object instantiated by passing a \doxyref{Scene\+Curve}{p.}{classSceneCurve} to its constructor and by an implementation of an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} which must be installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}. Once a \doxyref{Curve\+Item}{p.}{classCurveItem} is constructed with a \doxyref{Scene\+Curve}{p.}{classSceneCurve}, the \doxyref{Scene\+Curve}{p.}{classSceneCurve} {\itshape takes the ownership of the \doxyref{Curve\+Item}{p.}{classCurveItem}, destroying it when the curve is destroyed}. Once an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation instance is installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}, the \doxyref{Curve\+Item}{p.}{classCurveItem} {\itshape takes the ownership of the item painter} destroying it on \doxyref{Curve\+Item}{p.}{classCurveItem} destruction.
\end{DoxyParagraph}
\doxyref{Line\+Painter}{p.}{classLinePainter} is an example of an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation. \doxyref{Dots\+Painter}{p.}{classDotsPainter} is another example.

\begin{DoxyParagraph}{Why ownership of Curve\+Item and Line\+Painter is passed to Scene\+Curve and Curve\+Item respectively?}
This particular implementation allows you to create the \doxyref{Curve\+Item}{p.}{classCurveItem} and one or more \doxyref{Line\+Painter}{p.}{classLinePainter} objects and forget about their memory management. \end{DoxySeeAlso}
\begin{DoxyParagraph}{Some more details about item deletion.}
\begin{DoxyItemize}
\item {\itshape \doxyref{Curve\+Item}{p.}{classCurveItem}} is a Q\+Graphics\+Object and when it\textquotesingle{}s added to a Q\+Graphics\+Scene through Q\+Graphics\+Scene\+::add\+Item the ownership is taken by the Q\+Graphics\+Scene. This means that every \doxyref{Curve\+Item}{p.}{classCurveItem} is deleted on scene deletion, i.\+e. on \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} deletion, since the plot scene is a child of \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. \item {\itshape \doxyref{Scene\+Curve}{p.}{classSceneCurve}} is created by \doxyref{Plot\+Scene\+Widget\+::add\+Curve}{p.}{classPlotSceneWidget_ad84fa604445007630e530ad65b24b828} as a child of \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. Each curve is thus deleted on \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} destruction. \item {\itshape \doxyref{Line\+Painter}{p.}{classLinePainter}}, as in the {\itshape example above}, must be constructed by passing a \doxyref{Curve\+Item}{p.}{classCurveItem} in its constructor. {\itshape All \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementations}, such as \doxyref{Line\+Painter}{p.}{classLinePainter}, must inherit from Q\+Object and the parent must be a \doxyref{Curve\+Item}{p.}{classCurveItem}, as you can see in the \doxyref{Line\+Painter}{p.}{classLinePainter} implementation. In this way, on \doxyref{Curve\+Item}{p.}{classCurveItem} deletion, the \doxyref{Line\+Painter}{p.}{classLinePainter} is automatically deleted, as a child of \doxyref{Curve\+Item}{p.}{classCurveItem}.\end{DoxyItemize}

\end{DoxyParagraph}
\begin{DoxyParagraph}{Curve removal.}
You can remove a \doxyref{Scene\+Curve}{p.}{classSceneCurve} at runtime by calling Plot\+Scene\+Widget\+::remove\+Curve. This method will delete the associated \doxyref{Curve\+Item}{p.}{classCurveItem} (and all its painters, such as \doxyref{Line\+Painter}{p.}{classLinePainter}, for example) and then delete the \doxyref{Scene\+Curve}{p.}{classSceneCurve} itself. 
\end{DoxyParagraph}

This property sets the Viewport\+Update\+Mode to {\bfseries{Q\+Graphics\+View\+::\+No\+Viewport\+Update}} meaning that Q\+Graphics\+View will never update its viewport when the scene changes; the user is expected to control all updates. This mode disables all (potentially slow) item visibility testing in Q\+Graphics\+View, and is suitable for scenes that either require a fixed frame rate, or where the viewport is otherwise updated externally.

\doxyparagraph*{Note}

When you enable manual scene update, then you must call set\+Refresh\+Period with the desired refresh period of the scene, which typically must be equal to the fastest refresh rate needed by the fastest item. ~\newline
 Calling set\+Manual\+Scene\+Update with a false parameter, restores Q\+Graphics\+Scene default Viewport\+Update\+Mode, which is Q\+Graphics\+View\+::\+Minimal\+Viewport\+Update, according to the Qt documentation.


\begin{DoxyParams}{Parameters}
{\em manual} & true update is managed by an internal timer. set\+Refresh\+Period must be called with a greater than zero value in order to activate the timer and have the scene refreshed with that interval.\\
\hline
{\em manual} & false the update is set to Q\+Graphics\+View\+::\+Minimal\+Viewport\+Update, the default Qt setting for the Viewport\+Update\+Mode property. The internal refresh timer, if previously activated and started with a valid set\+Refresh\+Period method call is {\bfseries{automatically}} stopped for you. The timer is not deleted, it is just stopped, so that a subsequent call to set\+Manual\+Scene\+Update with a true parameter {\bfseries{automatically restarts the timer with the timeout previously set with set\+Refresh\+Period}}. In this way, a second manual mode activation does not need another call to set\+Refresh\+Period. A call to set\+Refresh\+Period with zero or a negative value instead produces the {\bfseries{ destruction}} of the timer.\\
