\subsection{Detailed Description}
An editor factory is a class that is able to create an editing widget of a specified type (e.\+g. line edits or comboboxes) for a given \doxyref{Qt\+Property}{p.}{classQtProperty} object, and it is used in conjunction with the \doxyref{Qt\+Abstract\+Property\+Manager}{p.}{classQtAbstractPropertyManager} and \doxyref{Qt\+Abstract\+Property\+Browser}{p.}{classQtAbstractPropertyBrowser} classes.

When using a property browser widget, the properties are created and managed by implementations of the \doxyref{Qt\+Abstract\+Property\+Manager}{p.}{classQtAbstractPropertyManager} class. To ensure that the properties\textquotesingle{} values will be displayed using suitable editing widgets, the managers are associated with objects of \doxyref{Qt\+Abstract\+Editor\+Factory}{p.}{classQtAbstractEditorFactory} subclasses. The property browser will use these associations to determine which factories it should use to create the preferred editing widgets.

Typically, an editor factory is created by subclassing the \doxyref{Qt\+Abstract\+Editor\+Factory}{p.}{classQtAbstractEditorFactory} template class which inherits \doxyref{Qt\+Abstract\+Editor\+Factory\+Base}{p.}{classQtAbstractEditorFactoryBase}. But note that several ready-\/made implementations are available\+:
