\subsection{Detailed Description}
A property browser is a widget that enables the user to edit a given set of properties. Each property is represented by a label specifying the property\textquotesingle{}s name, and an editing widget (e.\+g. a line edit or a combobox) holding its value. A property can have zero or more subproperties.

The top level properties can be retrieved using the \doxyref{properties()}{p.}{classQtAbstractPropertyBrowser_a64371ab43c9e8c9cecf7fb692a531e0e} function. To traverse each property\textquotesingle{}s subproperties, use the \doxyref{Qt\+Property\+::sub\+Properties()}{p.}{classQtProperty_aaf95c11630e0aae1f026a28cac7d5578} function. In addition, the set of top level properties can be manipulated using the \doxyref{add\+Property()}{p.}{classQtAbstractPropertyBrowser_a06c72eac8e19c78db1ea559d8df74868}, \doxyref{insert\+Property()}{p.}{classQtAbstractPropertyBrowser_abceee6a2a84e0b57dd6ae3837df95fe3} and \doxyref{remove\+Property()}{p.}{classQtAbstractPropertyBrowser_a50fd724c6ff83eadbca7665be970e350} functions. Note that the \doxyref{Qt\+Property}{p.}{classQtProperty} class provides a corresponding set of functions making it possible to manipulate the set of subproperties as well.

To remove all the properties from the property browser widget, use the \doxyref{clear()}{p.}{classQtAbstractPropertyBrowser_af1d62607f638d9f3592755cc90db3aaa} function. This function will clear the editor, but it will not delete the properties since they can still be used in other editors.

The properties themselves are created and managed by implementations of the \doxyref{Qt\+Abstract\+Property\+Manager}{p.}{classQtAbstractPropertyManager} class. A manager can handle (i.\+e. create and manage) properties of a given type. In the property browser the managers are associated with implementations of the \doxyref{Qt\+Abstract\+Editor\+Factory}{p.}{classQtAbstractEditorFactory}\+: A factory is a class able to create an editing widget of a specified type.

When using a property browser widget, managers must be created for each of the required property types before the properties themselves can be created. To ensure that the properties\textquotesingle{} values will be displayed using suitable editing widgets, the managers must be associated with objects of the preferred factory implementations using the \doxyref{set\+Factory\+For\+Manager()}{p.}{classQtAbstractPropertyBrowser_a952b5b393e83f16253033fa4a9f5ea89} function. The property browser will use these associations to determine which factory it should use to create the preferred editing widget.

Note that a factory can be associated with many managers, but a manager can only be associated with one single factory within the context of a single property browser. The associations between managers and factories can at any time be removed using the \doxyref{unset\+Factory\+For\+Manager()}{p.}{classQtAbstractPropertyBrowser_a55c476dd090c48b84927b7fbce3106d8} function.

Whenever the property data changes or a property is inserted or removed, the \doxyref{item\+Changed()}{p.}{classQtAbstractPropertyBrowser_a222c0033b0585cecf2a0717826f6e96b}, \doxyref{item\+Inserted()}{p.}{classQtAbstractPropertyBrowser_a62ec622f3c0381141a206441941b7e67} or \doxyref{item\+Removed()}{p.}{classQtAbstractPropertyBrowser_ad605abd88d30b906e85e920d64727a9e} functions are called, respectively. These functions must be reimplemented in derived classes in order to update the property browser widget. Be aware that some property instances can appear several times in an abstract tree structure. For example\+:

100\%   
\begin{DoxyCode}
QtProperty *property1, *property2, *property3;

property2->addSubProperty(property1);
property3->addSubProperty(property2);

QtAbstractPropertyBrowser *editor;

editor->addProperty(property1);
editor->addProperty(property2);
editor->addProperty(property3);
\end{DoxyCode}
  

The \doxyref{add\+Property()}{p.}{classQtAbstractPropertyBrowser_a06c72eac8e19c78db1ea559d8df74868} function returns a \doxyref{Qt\+Browser\+Item}{p.}{classQtBrowserItem} that uniquely identifies the created item.

To make a property editable in the property browser, the \doxyref{create\+Editor()}{p.}{classQtAbstractPropertyBrowser_ae138c214f5f5638c6750e357cd83c5c7} function must be called to provide the property with a suitable editing widget.

Note that there are two ready-\/made property browser implementations\+:

Connects the given {\itshape manager} to the given {\itshape factory}, ensuring that properties of the {\itshape manager\textquotesingle{}s} type will be displayed with an editing widget suitable for their value.

For example\+:


\begin{DoxyCode}
QtIntPropertyManager *intManager;
QtDoublePropertyManager *doubleManager;

QtProperty *myInteger = intManager->addProperty();
QtProperty *myDouble = doubleManager->addProperty();

QtSpinBoxFactory  *spinBoxFactory;
QtDoubleSpinBoxFactory *doubleSpinBoxFactory;

QtAbstractPropertyBrowser *editor;
editor->setFactoryForManager(intManager, spinBoxFactory);
editor->setFactoryForManager(doubleManager, doubleSpinBoxFactory);

editor->addProperty(myInteger);
editor->addProperty(myDouble);
\end{DoxyCode}


In this example the {\ttfamily my\+Integer} property\textquotesingle{}s value is displayed with a Q\+Spin\+Box widget, while the {\ttfamily my\+Double} property\textquotesingle{}s value is displayed with a Q\+Double\+Spin\+Box widget.

Note that a factory can be associated with many managers, but a manager can only be associated with one single factory. If the given {\itshape manager} already is associated with another factory, the old association is broken before the new one established.

This function ensures that the given {\itshape manager} and the given {\itshape factory} are compatible, and it automatically calls the \doxyref{Qt\+Abstract\+Editor\+Factory\+::add\+Property\+Manager()}{p.}{classQtAbstractEditorFactory_af32e5e5341a42399841c97c0e82f1f5c} function if necessary.
