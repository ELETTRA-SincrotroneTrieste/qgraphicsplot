\begin{DoxyParagraph}{Concepts.}
Each curve belongs to a \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} and has two axes associated. The \doxyref{Scene\+Curve}{p.}{classSceneCurve} implements the \doxyref{Axis\+Change\+Listener}{p.}{classAxisChangeListener} interface in order to know when the axes bounds change and the axis auto\+Scale property changes.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Curve drawing.}
The \doxyref{Scene\+Curve}{p.}{classSceneCurve} does not draw anything on the scene. Its just recollects data and keeps a list of points whose coordinates map the real points into the plot coordinate system. In order to draw, you must create a \doxyref{Curve\+Item}{p.}{classCurveItem} and then attach it to the \doxyref{Scene\+Curve}{p.}{classSceneCurve}, normally constructing a \doxyref{Curve\+Item}{p.}{classCurveItem} with the \doxyref{Scene\+Curve}{p.}{classSceneCurve} as parameter.
\end{DoxyParagraph}

\begin{DoxyParagraph}{Item painters ownership}
The \doxyref{Curve\+Item}{p.}{classCurveItem} takes the ownership of every installed \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation instance. This means that all the installed \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} objects are deleted on \doxyref{Curve\+Item}{p.}{classCurveItem} deletion.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Curve\+Item ownership}
If you call Plot\+Scene\+Widget\+::remove\+Curve, then the associated \doxyref{Curve\+Item}{p.}{classCurveItem} is deleted. If you delete a \doxyref{Scene\+Curve}{p.}{classSceneCurve} without invoking Plot\+Scene\+Widget\+::remove\+Curve (bad), then the associated \doxyref{Curve\+Item}{p.}{classCurveItem} is deleted again for you. This means that once installed on a \doxyref{Scene\+Curve}{p.}{classSceneCurve}, the ownership of the \doxyref{Curve\+Item}{p.}{classCurveItem} is taken by the \doxyref{Scene\+Curve}{p.}{classSceneCurve}. With the term {\itshape install} we mean the
\end{DoxyParagraph}

\begin{DoxyParagraph}{Obtaining the Curve\+Item Q\+Graphics\+Object objects associated to the Scene\+Curve}
An internal reference to the (unique) \doxyref{Curve\+Item}{p.}{classCurveItem} object is kept in order to be able to provide it to the users of \doxyref{Scene\+Curve}{p.}{classSceneCurve}. The method \doxyref{curve\+Item()}{p.}{classSceneCurve_ace76c4a9f25717c47e3d9b9c22fa1c4e} can be used to retrieve a reference to the unique \doxyref{Curve\+Item}{p.}{classCurveItem} attached to the curve. For instance to change the colour of the line at any time, having a reference to a \doxyref{Scene\+Curve}{p.}{classSceneCurve}\+:
\end{DoxyParagraph}

\begin{DoxyCode}{0}
\DoxyCodeLine{LinePainter *currentLinePainter = curve-\/>curveItem()-\/>linePainter();}
\DoxyCodeLine{currentLinePainter-\/>setLineColor(Qt::blue);}
\end{DoxyCode}


\begin{DoxyNote}{Note}
The code above only works if a unique \doxyref{Line\+Painter}{p.}{classLinePainter} was installed, just like in the {\itshape Example} in the previous paragraph. If a \doxyref{Line\+Painter}{p.}{classLinePainter} coexists with a \doxyref{Dots\+Painter}{p.}{classDotsPainter}, for instance, you need to carefully read the note below.

Multiple \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation instances can be installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}. If they all are Q\+Object-\/derived, then a Qt property can be used to distinguish between different painters. For instance\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{linePainter-\/>setProperty(\textcolor{stringliteral}{"{}type"{}}, \textcolor{stringliteral}{"{}Lines"{}});}
\end{DoxyCode}

The \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} can be statically casted to the correct class type. 
\end{DoxyNote}

\begin{DoxyParagraph}{The alias property.}
The Qt {\itshape alias} property can be set so that some objects can use it to display the curve name instead of the string returned by \doxyref{Scene\+Curve\+::name}{p.}{classSceneCurve_a5e7482ea4d79c6abfa889cc42a28beca} method.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Note\+: NaN values}
The NaN y values are mapped on the same y as the previous y value. If the first point of a curve is NaN, then the lower bound is chosen for the NaN representation.
\end{DoxyParagraph}
{\bfseries{Important}}\+: it is up to the {\bfseries{painter}} to evidentiate a NaN value. This must be done inspecting the y data value of a point.
